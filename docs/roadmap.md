## Phase 1：核心功能 MVP（Week 2~4）

**目标**：PDF 转换 + 编辑 + 压缩 三大核心可用

### Week 2：PDF 编辑模块（最简单，最先做）

| 功能 | 说明 | 技术方案 |
|------|------|----------|
| PDF 合并 | 选择多个 PDF → 合并成一个 | `pypdf.PdfWriter.append()` |
| PDF 拆分 | 按页拆分 / 按范围拆分 | `pypdf` 逐页输出 |
| 提取页面 | 选择页码范围 → 导出新PDF | `pypdf` 页面操作 |
| 删除页面 | 选择页码 → 删除后保存 | `pypdf` 页面操作 |
| 旋转页面 | 90°/180°/270° 旋转 | `pypdf.PdfWriter.pages[].rotate()` |
| 页面排序 | 拖拽调整页面顺序 | `pypdf` 重组 |

### Week 3：PDF 转换模块（核心竞争力）

| 功能 | 说明 | 技术方案 |
|------|------|----------|
| PDF → Word | 保留格式、表格 | `pdf2docx.Converter` |
| PDF → Excel | 提取表格到Excel | `pdfplumber` + `openpyxl` |
| PDF → 图片 | 逐页渲染为PNG/JPG | `pdfplumber` 的 `to_image()` |
| PDF → HTML | 保留格式转网页 | `pdfminer` 基础转换 |
| 图片 → PDF | 多张图合并/单图转PDF | `reportlab` / `Pillow` |

### Week 4：PDF 压缩模块

| 功能 | 说明 | 技术方案 |
|------|------|----------|
| 无损压缩 | 移除冗余数据、优化结构 | `pypdf` 的 `compress_content_streams()` |
| 有损压缩 | 降低图片质量 | `pypdf` 遍历图片对象 → `Pillow` 压缩 → 写回 |
| 自定义 | 用户自定义图片质量(1~100) | 滑块调节 |

### Week 4 附赠：文件列表增强

- [ ] 批量拖拽添加文件（支持文件夹递归）
- [ ] 文件列表多选、全选、排序
- [ ] 显示文件信息（页数、大小、日期）
- [ ] 处理进度条（多线程）
- [ ] 处理完成后「打开文件夹」

## Phase 3：激活系统 + 打包（Week 7）

**目标**：完成商业化最后一步

### 激活系统

| 任务 | 说明 |
|------|------|
| 机器码生成 | 读取 CPU + 主板 + MAC → MD5 |
| RSA密钥对 | 生成公钥私钥，公钥放软件里，私钥你自己保留 |
| 激活码生成器 | 你本地运行，输入机器码 → 输出激活码 |
| 激活验证 | 软件启动时验签 + 比对机器码 |
| 激活状态存储 | 加密存储到本地文件/注册表 |
| 激活码管理后台 | 简易网页或CSV记录（谁买了、激活码、激活日期） |

## 开发原则

1. **先跑通，再优化**：功能先能用，界面先整齐，发布后再迭代
2. **每天 commit**：哪怕只写了 50 行代码，也要记录
3. **MVP 思维**：Week 4 结束时，合并+拆分+转换+压缩这 4 个功能必须能完整使用，其他都是加分项
4. **用 AI 加速**：重复代码、样式调整、正则处理等让 AI 写，你专注核心逻辑
